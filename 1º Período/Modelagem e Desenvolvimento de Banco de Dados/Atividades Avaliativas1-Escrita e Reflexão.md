##  ✅ Resposta:

Pelo que o vídeo abordou e me baseando na minha vivencia acadêmica e profissional da área de sistemas de informação, posso dizer que criar um banco de dados exige planejamento cuidadoso para evitar falhas e garantir que os dados realmente apoiem decisões estratégicas.

O primeiro procedimento é a modelagem de dados, que organizando as informações em entidades e relacionamentos, reduzem redundâncias e inconsistências, segundo LIMA; LOPES; NASCIMENTO (2026). O segundo passo é a normalização, aplicando regras formais para estruturar tabelas de forma eficiente, assegurando a integridade e consistência, conforme ALURA (2023). E por fim, a implementação de backup e restauração minimizando riscos de perda de dados e assegurando a continuidade operacional, agregando valor ao negócio, conforme DATABRICKS (2024).

Referências:
- LIMA, E. L. F.; LOPES, P. H. M. B.; NASCIMENTO, F. A. Boas práticas de modelagem de dados em bancos relacionais: uma revisão rápida de literatura. Revista Sociedade Científica, v. 9, n. 1, p. 1188-1202, 2026.
- DATABRICKS. Modelagem de Banco de Dados: Técnicas, Processo e Melhores Práticas. Blog Databricks, 2024.
- ALURA. Normalização em Banco de Dados - Estrutura. Publicado em 06 mar. 2023.





## Sobre o vídeo

O vídeo “10 Conceitos de Bancos de Dados que todo estudante precisa conhecer” apresenta uma lista dos principais fundamentos de bancos de dados que todo estudante de tecnologia deve dominar.

📚 Principais Conceitos de Bancos de Dados
Modelagem de Dados: Explica os três níveis de modelagem: conceitual, lógica e física. Essencial para estruturar corretamente informações.
Modelo Relacional: Base da maioria dos sistemas de banco de dados. Organiza dados em tabelas com linhas e colunas.
Normalização: Processo de reduzir redundâncias e inconsistências, aplicando formas normais (1FN, 2FN, 3FN).
Chaves Primárias e Estrangeiras: Garantem integridade referencial e relacionamentos entre tabelas.
SQL: Linguagem padrão para manipulação de dados: SELECT, JOIN, UPDATE, DELETE.
NoSQL: Bancos não relacionais, usados para dados não estruturados e escalabilidade.
Transações: Operações que seguem as propriedades ACID (Atomicidade, Consistência, Isolamento, Durabilidade).
Índices: Melhoram a performance de consultas, mas exigem planejamento para não prejudicar inserções.
Segurança de Dados: Controle de acesso, criptografia e boas práticas para proteger informações.
Backup e Recuperação: Estratégias para garantir disponibilidade e integridade em caso de falhas.

🎯 Observações Importantes
O vídeo é produzido pelo canal Boson Treinamentos, bastante conhecido por conteúdos de TI.
Ele serve como introdução, não como curso completo — cada conceito pode ser aprofundado em aulas específicas.
Há menções a cursos pagos e materiais complementares, mas o foco principal é dar uma visão geral dos pilares de bancos de dados.
